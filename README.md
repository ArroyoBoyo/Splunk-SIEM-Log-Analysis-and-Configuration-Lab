# Splunk-SIEM-Log-Analysis-and-Configuration-Lab
A foundational SIEM project focused on Splunk setup, log ingestion, dashboard creation, and understanding how system, network, and security logs are collected and analyzed in a SOC environment.
# 🎯 Objective
Build and configure a foundational Splunk SIEM lab on Windows that ingests multiple log sources, demonstrates understanding of key system and security events, and presents that data through searches, dashboards, and a report.
# 🛠️ Tools I used
- ✅ Splunk Enterprise and Universal Forwarder
- ✅ Windows Event Logs
- ✅ Sysmon
- ✅ Firewall logs
- ✅ Network/system logs
# 🚧 What I Built
I installed and configured Splunk Enterprise in a Windows-based VM environment, ingesting multiple log sources and validating their searchability to understand how Splunk centralizes and enables review of security events.
# 🔍 What I Analyzed
This lab focused on understanding log collection, event context, and investigative use cases by exploring Windows Event Logs, Sysmon data, and firewall/network logs to gain visibility into authentication, system activity, and security-relevant events.
# 📖 What the Outcome Was
This project resulted in a functional Splunk lab covering SIEM setup, log ingestion, searching, and dashboards, providing a foundation for future detection and incident response labs.
