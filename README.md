# Splunk-SIEM-Log-Analysis-and-Configuration-Lab
This project involved configuring Splunk Enterprise, exploring and analyzing Windows event logs, creating a basic alert for suspicious failed login activity, and improving visibility into authentication and system events in a SOC-style workflow.
# 🛠️ Tools I used
- ✅ Splunk Enterprise
- ✅ Windows 11 VM
- ✅ Windows Security Logs
- ✅ Windows Application Logs
- ✅ Windows System Logs
# 🚧 What I Built
- I configured a basic Splunk SIEM lab in a Windows VM environment and connected log data for ingestion into Splunk. 
- I then used Splunk to review Windows event activity, identify important log categories, and explore how SIEM platforms centralize information for security monitoring.
# 🔍 What I Analyzed
This project focused on understanding how Windows Security logs appear in Splunk and how they can be used for monitoring suspicious behavior. I specifically worked with:
- Failed logon activity
- Repeated authentication failures
- Process creation visibility
- Basic security event monitoring
# Alerting and Monitoring
As part of the lab, I created a basic alert for suspicious failed login activity using relevant Windows Event Codes. This demonstrated how specific event types can be queried, monitored, and simplified via alert logic inside Splunk.
# 📖 What the Outcome Was
This project resulted in a functioning foundational SIEM lab that demonstrates:
- Splunk setup and configuration
- Windows log ingestion
- Basic log querying and analysis
- Understanding of key security event types
- Alert creation for suspicious failed logon activity
- Dashboard creation for monitoring visibility
